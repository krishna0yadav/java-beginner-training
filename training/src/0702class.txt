Comparable vs Comparator [imp]
------------------------------
-> It is used to sort object data.


Comparable inteface
-> it implements by String and all Wrapper class.[by default]
-> in other language,It implements by predefined classes.
-> Data must be homogeneous.
-> Data must implements by Comparable inteface.
-> Mixing business logic and sorting logics.
-> Only one property sorting is allowed.
-> Comparable belongs to java.lang package.
-> we use method Collections.sort(Object)
Case1:
-> Example:
	Collections.sort() is used to sort collection.
	it internally uses int compareTo() method.

-> How compareTo() method works
	
v1			v2				result
-----------------------------------
"Ajay"	   "Ajay"			 0			no change
"Ajay"	   "Vijay"          -ve         no change
"Vijay"    "Ajay"           +ve         change    

--------------------------------------------------------
Case2:
Hetrogeneous Data
Example:
"mark" and 10

-> Comparison of two different object is not possible.
-> if compare by compreTo() , we will get ClassCastException.

Comparator Interface
----------------------
-> 	We can seperate business logics and sorting logics in different classes.
-> 	Multiple properties sotring is allowed.
-> 	it belongs to java.util package.
->  We can use Collections.sort(Object,Object);
-> 	Predefined classes are not implementing Comparator
->  Comparator only for userdefined classes.

Example:

Multithreading
---------------
-> It is a process of executing multiple thread simultaneosly.
-> it used in games, animations, server programming etc.

Thread
------
-> It is a light weigth sub process.
-> It is a smallest unit of processing.
-> It is a seperate path of execution.
-> Threads are independent.
-> if there occurs exception in one thread, it doesn't affect other threads.
-> It shares common memory area.
-> At a time one thread is executed only.

Thread Life Cycle
------------------
New
Running
Suspended
Blocked
Terminated

Save in image

Thread Creation
---------------
there are two ways to create thread

1. By extending Thread class.
2. By implementing Runable interface.

By extending Thread class
--------------------------
import java.util.*;

public class Test extends Thread {
	
	public void run()
	{
		System.out.println("Thread is running");
	}
	public static void main(String[] args)
	{ 	
		Test t1 = new Test();
		t1.start();
	}
}

By implementing Runnable interface.
import java.util.*;

public class Test implements Runnable {
	
	public void run()
	{
		System.out.println("Thread is running");
	}
	public static void main(String[] args)
	{ 	
		Test t1 = new Test();
		Thread th = new Thread(t1);
		
		th.start();
	}
}
